Game AI Summit

SCALLS: Developing Generalist AI Agents Using Video Games

Alexander Murlakov | Google DeepMind

Google DeepMind shares SCALLS - a Scalable, Instructable, Multi-World Agent that can execute natural language instructions across diverse video game environments. This project explores using games as a training ground for developing general AI.

+
Overview / 演讲概述

SCALLS (Scalable, Instructable, Multi-World Agent) is a groundbreaking project from Google DeepMind. The agent demonstrates not only excellent performance in individual games but crucially possesses cross-game generalization capabilities - skills learned in Astroneer can transfer to Simulation Farm or Happy Farm. The speaker detailed the system's technical architecture, including how the visual encoder, language understanding module, and action execution layer work together.

SCALLS（可扩展指令多世界代理）是谷歌深度思维推出的开创性项目。该代理不仅能在单一游戏中表现出色，更关键的是具备跨游戏的泛化能力——在《异星探险家》中学到的技能可以迁移到《模拟农场》或《开心农场》中。

Core Technical Highlights / 核心技术亮点
  • SCALLS Architecture: Vision-language-action integrated design based on large language models / 基于大语言模型的视觉-语言-动作一体化设计
  • Cross-Game Generalization: AI trained in one game can transfer to other games / 在一个游戏中训练的AI可以迁移到其他游戏中
  • World Knowledge Embedding: Game state information encoded into the agent's reasoning process / 将游戏世界的状态信息编码到代理的推理过程中
  • Progressive Training: From simple instructions to complex tasks / 从简单指令到复杂任务的渐进式训练策略
Industry Impact / 行业影响

The SCALLS project demonstrates that games can serve as ideal environments for training generalist AI agents. This "games as training ground" methodology is being adopted by more research institutions and enterprises, promising to advance the development of artificial general intelligence.

斯卡莉项目证明了游戏可以作为训练通用人工智能代理的理想环境。这种"以游戏为训练场"的方法论正在被越来越多的研究机构和企业采纳，有望推动通用人工智能的发展。

"Games provide an ideal controlled, quantifiable, and reproducible environment for AI research. The diversity of game worlds makes them perfect for training generalist agents."

Building a Statistical Analysis-Based Anti-Cheat Framework for For Honor

Bettina Hein | Ubisoft Entertainment

Ubisoft demonstrates how to build anti-cheat frameworks using statistical analysis and machine learning. By analyzing player behavior data patterns, they achieve automated detection and classification of cheating behavior.

+
Overview / 演讲概述

Cheat detection is a common challenge for all online competitive games. Traditional signature-based anti-cheat methods are easily circumvented. Ubisoft proposed a new approach based on behavioral statistical analysis. By collecting extensive data on normal players and known cheaters, they built anomaly detection models that can identify abnormal behavior without relying on specific cheat tool signatures.

作弊检测是所有在线竞技游戏面临的共同挑战。传统的基于签名的反作弊方法容易被规避，而育碧提出了一种基于行为统计分析的新方法。

Key Innovation / 核心创新

The system's key innovation lies in its "progressive confidence" mechanism - instead of banning players for one suspicious behavior, it builds reliability through continuous observation and cumulative evidence, achieving the best balance between anti-cheat effectiveness and player experience.

该系统的关键创新在于其"渐进式置信度"机制——不会因为一次可疑行为就封禁玩家，而是通过持续观察和累积证据来提高判断的可靠性。

Game AI Summit

A History of Game AI: From Rule Systems to Deep Learning

Yoichiro Miiike | Square Enix

A panoramic review of game AI development history from Square Enix. Systematically tracing the technological evolution from early finite state machines to behavior trees, then to deep learning and generative AI.

+
Overview / 演讲概述

Yoichiro Miiike is a veteran AI researcher in the Japanese game industry. This talk is not only a review of technological evolution but also deep thinking about game design philosophy - AI technology advances don't mean the obsolescence of old methods, but provide designers with a richer toolkit.

三池洋一郎是日本游戏行业资深的AI研究者。这场演讲不仅是对技术演进的回顾，更是对游戏设计哲学的深度思考。

Core Insights / 核心观点

"Many classic AI technologies (such as finite state machines, behavior trees) remain indispensable in modern games. Their advantages lie in predictability and controllability. Deep learning and generative AI are more suitable for 'fuzzy decision-making' and 'behavioral diversity' scenarios. The best practice is to choose the right tool for the specific need, rather than blindly pursuing new technology."

演讲者特别强调，许多经典AI技术在现代游戏中仍然不可或缺，它们的优势在于可预测性和可控性。深度学习和生成式AI更适合用于"模糊决策"和"行为多样性"的场景。

"Database-Oriented Design": Why We Built Our MMORPG Inside a Database

Tyler Cloutier | Clockwork Labs (SpacetimeDB)

Exploring why Clockwork Labs built their MMORPG backend entirely inside a database, sharing simplified development experience with RDBMS + WebAssembly.

+
Overview / 演讲概述

Traditional multiplayer game backends typically involve game servers, databases, message queues, and multiple components. But Clockwork Labs chose a bold approach: running the entire game backend - including game logic, state management, and network synchronization - entirely inside a database.

传统多人在线游戏的后端架构通常涉及游戏服务器、数据库、消息队列等多个组件，但钟表实验室选择了一个大胆的方案：将整个游戏后端——包括游戏逻辑、状态管理和网络同步——全部运行在数据库内部。

Technical Innovation / 技术创新
  • SpacetimeDB: A temporal database engine designed for games / SpacetimeDB：一款为游戏设计的时间数据库引擎
  • All game logic runs as WebAssembly modules inside the database / 所有游戏逻辑作为WebAssembly模块在数据库内运行
  • Automatic state synchronization across all connected clients / 所有连接客户端自动状态同步
  • Dramatically simplified backend architecture / 大幅简化的后端架构

"By putting the game inside the database, we eliminated the entire category of consistency bugs between server state and database state. The database is the truth."
